## 실습 1 요구사항

**목표: `useImmer`를 활용한 장바구니 수량 조절 기능 구현**

**1. 데이터 구조 및 상태 관리 (State)**
- `CartItem` 인터페이스를 정의합니다: `id(number)`, `name(string)`, `price(number)`, `quantity(number)`.
- `useState` 대신 **`useImmer`** 훅을 사용하여 초기 상품 목록(기계식 키보드, 게이밍 마우스 등)을 상태로 선언합니다.
- (옵션) `inventory` 항목을 추가한 후, 상품 수량이 재고를 초과하지 못하도록 설정합니다.

**2. 수량 변경 로직 구현 (Handler)**
- 상품의 `id`와 변화량(`delta`)을 인자로 받는 `updateQuantity` 함수를 작성합니다.
- **`updateCart` (Immer의 setter)** 내부에서 `draft.find()`를 사용하여 수정할 아이템을 찾습니다.
- **방어 로직**: 상품의 수량이 `1` 미만으로 내려가지 않도록 업데이트합니다. (옵션: 재고 초과도 방어 필요!)

**3. UI 구성 및 이벤트 바인딩**
- `cart.map()`을 사용하여 상품 목록을 렌더링합니다.
- **증가 버튼**: 클릭 시 `updateQuantity(id, 1)`을 호출합니다.
- **감소 버튼**: 클릭 시 `updateQuantity(id, -1)`을 호출하되, 현재 수량이 `1`인 경우 실행되지 않도록 조건부 처리를 합니다.

**4. 접근성(Accessibility) 및 사용자 경험**
- 각 버튼에 `aria-label`을 부여하여 스크린 리더 사용자가 용도를 알 수 있게 합니다. (예: `aria-label="기계식 키보드 수량 감소"`)
- 수량이 `1`일 때 감소 버튼에 `aria-disabled="true"` 속성을 적용하고 시각적으로 비활성화 상태임을 알립니다.
- 가격 정보는 `toLocaleString()`을 사용하여 천 단위 구분 쉼표를 표시합니다.

## 실습 2 요구사항

**목표: 조건부 렌더링을 활용한 삭제 및 복구 기능 구현**

**1. 개별 상품 삭제 핸들러 (`deleteItem`)**
- 삭제할 상품의 `id`를 인자로 받아 해당 아이템을 배열에서 제거합니다.
- **Immer 활용**: 삭제할 아이템의 인덱스를 찾아 직접 배열을 수정합니다.
- **사용자 경험**: 삭제 전 `confirm()`을 사용하여 사용자에게 다시 한번 확인 절차를 거칩니다.
- **JSX 마크업**: 삭제 버튼의 클래스 이름은 `S.deleteButton` 입니다.

**2. 장바구니 비우기 및 복구 핸들러 (`handleClear`, `handleRestore`)**
- **비우기**: `updateCart`를 호출하여 상태를 빈 배열(`[]`)로 업데이트합니다.
- **복구**: 초기 데이터인 `INITIAL_CART`를 상태에 다시 할당하여 목록을 복구합니다.
- **JSX 마크업**: 비우기/복구 버튼의 클래스 이름은 `S.transitionButton` 입니다.

**3. 파생된 상태 및 조건부 렌더링**
- **변수 활용**: `cart.length`를 기반으로 장바구니가 비어있는지 확인하는 변수(`hasCartItems`)를 만듭니다.
- **UI 분기**: 
    - 장바구니에 아이템이 있을 때는 **'전체 비우기'** 버튼을 보여줍니다.
    - 장바구니가 비었을 때는 **'복구하기'** 버튼을 보여줍니다.
    - 장바구니가 비었을 때는, `장바구니가 비어 있습니다.` 문구를 화면에 출력합니다. (`.emptyMessage`)

**4. UI 및 접근성**
- 삭제 버튼에 `aria-label`을 적용하여 어떤 상품이 삭제되는지 명시합니다. (예: `aria-label="기계식 키보드 삭제"`)

## 실습 3 요구사항

**목표: 파생 상태(Derived State)를 활용한 선택 제어 및 금액 합산 구현**

**1. 데이터 구조 확장 및 개별 선택 (`toggleItem`)**
- `CartItem` 인터페이스에 `checked: boolean` 속성을 추가합니다.
- `id`를 인자로 받아 해당 아이템의 `checked` 상태를 반전(`!`)시키는 함수를 작성합니다.
- **UI**: 각 아이템의 체크박스와 상품 정보(`label`)를 `htmlFor`로 연결하여 텍스트 클릭 시에도 체크되도록 접근성을 개선합니다.
- **JSX 마크업**: 추가할 `<input>` 요소의 클래스 이름은 `.checkbox`입니다.

**2. 전체 선택 및 해제 (`handleToggleAll`)**
- **JSX 마크업**: `.heaader>#all-check.checkbox+label[for="all-check"]` 구조 작성 후 '전체 선택 (상품 개수)'를 출력합니다.
- **로직**: 현재 '전체 선택 상태'를 반전시켜 모든 아이템의 체크 값을 일괄 변경합니다.
- **Immer 활용**: Immer를 사용하여 모든 요소의 속성을 직접 수정하는 간결한 코드를 작성합니다.

**3. 효율적인 파생 변수 계산 (Logic)**
- 별도의 상태(`useState`)를 만들지 않고, 렌더링 시점에 다음 변수들을 계산합니다.
    - **`isAllChecked`**: 장바구니가 비어있지 않고, 모든 아이템의 체크 상태를 확인합니다.
    - **`totalPrice`**: 체크된 아이템들만 필터링하여 총합을 구합니다.

**4. UI 피드백 및 접근성**
- **JSX 마크업**: `.footer>span.totalLabel+strong.totalPrice` 구조 작성 후 '결제 예정 금액: 총액(원)'을 출력합니다.
- **금액 표시**: `totalPrice.toLocaleString()`을 사용하여 화폐 단위를 가독성 있게 출력합니다.
- **접근성**: 총 금액이 변경될 때마다 스크린 리더가 읽어줄 수 있도록 합계 영역에 `aria-live="polite"` 속성을 적용합니다.
- **상태 전이**: 장바구니가 비었을 때 안내 문구와 함께 '복구하기' 버튼이 나타나도록 조건부 렌더링을 완성합니다.
