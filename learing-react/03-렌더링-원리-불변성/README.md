# 핵심 목표

**DAY 03. 렌더링 원리와 불변성**

- **리액트 렌더링 파이프라인과 가상 DOM의 이해**  
`Trigger` → `Render` → `Commit`으로 이어지는 리액트의 렌더링 메커니즘을  
주문과 서빙 비유를 통해 명확히 이해하고, 가상 DOM이 실제 DOM과 상호작용하며   
UI를 업데이트하는 최적화 원리를 체득합니다.
- **불변성(Immutability) 유지와 데이터 관리**  
리액트가 상태 변화를 감지하는 기준인 '참조 동일성'의 개념을 파악하고,  
전개 연산자(`...`)를 활용해 원본 데이터를 훼손하지 않고 객체와 배열을  
안전하게 CRUD(생성·조회·수정·삭제)하는 불변성 유지 능력을 기릅니다.
- **재조정(Reconciliation)과 Key 변경 패턴의 활용**  
컴포넌트의 재사용과 초기화를 결정하는 `key` 속성의 올바른 사용법을 익히고,  
단순히 데이터 식별을 넘어 `key` 변경을 통해 컴포넌트 상태를 강제로 초기화하는  
실무적인 상태 제어 패턴을 습득합니다.
- **디버깅 루틴 수립과 렌더링 최적화 기초**  
React 개발도구(DevTools)를 활용해 컴포넌트 트리와 State/Props의 변화를  
추적하고, 불필요한 렌더링이 발생하는 원인을 분석하여 성능 최적화를 위한  
논리적인 디버깅 루틴을 확립합니다.
