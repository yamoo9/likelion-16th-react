# 핵심 목표

**Context API를 활용한 전역 상태 관리**

- **Props Drilling 문제 해결과 Context의 필요성 체득**  
컴포넌트 계층이 깊어질 때 발생하는 Props Drilling의 불편함을 직접 경험하고,  
이를 해결하기 위한 Context API의 역할과 `createContext`, `Provider`,  
`useContext`의 유기적인 동작 원리를 완벽히 이해합니다.
- **효율적인 전역 상태 설계 및 Provider 배치 전략**  
애플리케이션 전체에서 공유해야 할 값(인증, 테마)과 컴포넌트 내부 상태를 명확히  
구분하고, 불필요한 리렌더링을 방지하기 위한 최적의 Provider 위치 선정 및  
데이터 구조 설계 능력을 기릅니다.
- **실무 중심의 전역 상태 활용 (테마·인증·모달 UI)**  
다크 모드(테마), 사용자 인증(Login/Logout), 전역 모달 다이얼로그 등  
실무 필수 기능을 Context로 직접 구현하며, 상태와 로직이 분리된  
유지보수성이 높은 클린 코드를 작성하는 역량을 확보합니다.
