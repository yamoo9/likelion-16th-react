## 실습 1. Context API와 Portal을 활용한 전역 모달 시스템

**목표**: 리액트 19, 웹 표준 애니메이션, 웹 접근성(A11y)을 고려한 전역 모달 구현

1.  **타입 정의 및 Context 생성**
    - `ModalData` 인터페이스: 모달의 제목(`title`)과 렌더링할 내용(`content`)을 정의합니다.
    - `ModalContextType` 타입: 외부에서 호출할 `open`, `close` 함수 타입을 정의합니다.
    - `ModalContext`를 생성하고 초기값은 `null`로 설정하여 안전한 사용을 유도합니다.

2.  **모달 상태 및 포커스 관리**
    - `modal` 상태로 데이터 존재 여부를 관리하고, `isClosing` 상태로 퇴장 애니메이션을 제어합니다.
    - `useRef`를 활용해 모달이 열리기 전 마지막으로 포커스된 요소를 저장하고, 닫힐 때 복구합니다.
    - `useId`를 사용하여 스크린 리더를 위한 고유한 `aria-labelledby` ID를 생성합니다.

3.  **리액트 19 최적화 및 애니메이션 처리**
    - `startTransition`을 사용하여 모달 렌더링의 우선순위를 조정, 부드러운 UI 전환을 보장합니다.
    - `onTransitionEnd` 이벤트를 활용해 CSS 애니메이션이 완전히 끝난 후 DOM에서 모달을 제거하는 로직을 구현합니다.

4.  **웹 접근성(A11y) 및 Side Effect 구현**
    - `useEffect` 내에서 모달 오픈 시 배경 스크롤을 차단(`overflow: hidden`)합니다.
    - `Focus Trap` 로직을 구현하여 Tab 키 이동이 모달 내부에서만 순환하도록 제어합니다.
    - ESC 키를 눌렀을 때 모달이 닫히도록 키보드 이벤트를 등록합니다.

5.  **Portal 렌더링 및 커스텀 훅**
    - `createPortal`을 사용하여 모달 UI를 `document.body` 바로 아래로 전송(Teleport)합니다.
    - `useModal` 커스텀 훅을 작성하여 컴포넌트 어디서든 간편하게 모달을 제어할 수 있게 합니다.
    - `main.tsx`에서 `ModalProvider`를 감싸 전역에서 모달 기능을 활성화합니다.

## 실습 2. 전역 모달 시스템을 활용한 비동기 액션 구현

**목표**: `useModal` 훅과 비동기 로직을 결합하여 실제 서비스와 유사한 삭제 흐름 구현

### **1. 커스텀 훅 및 상태 설정**
- **useModal 활용**: `useModal` 훅을 호출하여 `open`과 `close` 함수를 가져옵니다.
- **삭제 상태 관리**: `isDelete` 상태를 만들어 삭제 성공 후의 UI 전환(조건부 렌더링)을 처리합니다.

### **2. 비동기 삭제 API 시뮬레이션**
- **wait 유틸리티**: `wait(1500)`를 사용하여 네트워크 지연 상황을 시뮬레이션합니다.
- **상태 업데이트**: 삭제 작업이 성공적으로 완료되면 `setIsDelete(true)`를 호출하여 화면에 성공 메시지를 표시합니다.

### **3. 모달 호출 및 컴포넌트 주입**
- **handleOpenDeleteModal 구현**: 버튼 클릭 시 `open` 함수를 실행합니다.
- **콘텐츠 구성**: 모달의 본문으로 이전에 만든 `ConfirmDelete` 컴포넌트를 전달합니다.
- **Props 바인딩**: 
    - `onConfirm`: 위에서 정의한 `deletePostApi` 비동기 함수를 전달합니다.
    - `onCancel`: `useModal`에서 가져온 `close` 함수를 전달하여 취소 시 모달이 닫히도록 설정합니다.

### **4. 사용자 경험(UX) 강화**
- **조건부 렌더링**: 삭제 전에는 포스트 내용과 삭제 버튼을 보여주고, 삭제 후에는 성공 안내 문구를 보여주도록 JSX를 구성합니다.
- **CSS 모듈 적용**: `S` 객체를 사용하여 `shadcn/ui` 스타일이 적용된 레이아웃과 버튼 디자인을 완성합니다.
