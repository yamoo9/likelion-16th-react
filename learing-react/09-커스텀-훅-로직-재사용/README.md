# 핵심 목표

**커스텀 훅(Custom Hooks)과 로직 재사용**

- **훅의 설계 원칙과 로직의 재사용성 확보**  
리액트 훅의 호출 규칙과 클로저 기반의 의존성 원리를 깊이 있게 이해하고,  
반복되는 입력(`useInput`) 및 상태 제어(`useToggle`) 로직을 커스텀 훅으로  
분리하여 코드의 가독성과 재사용성을 극대화합니다.
- **비동기 데이터 처리의 고도화와 안정성 강화**  
`fetch` API와 `AbortController`를 결합한 전역 데이터 가져오기 훅(`useFetch`)을  
설계하고, 요청 취소(Abort) 및 경쟁 상태(Race Condition) 해결을 통해  
네트워크 통신 시 발생할 수 있는 데이터 꼬임 현상을 원천 차단합니다.
- **실전 비동기 패턴 적용 및 사용자 경험(UX) 최적화**  
영화 검색 실습을 통해 디바운스(Debounce)를 활용한 효율적인 API 호출,  
로딩/에러/성공 UI 분리, 실패 시 재시도(Refetch) 및 추가 로드(Pagination) 기능을  
구현하여 실무 수준의 완성도 높은 비동기 애플리케이션 구축 능력을 체득합니다.
