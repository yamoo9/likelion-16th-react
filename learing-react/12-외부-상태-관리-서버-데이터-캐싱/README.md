# 핵심 목표

**전역 상태 관리와 선언적 데이터 페칭**

- **Zustand를 활용한 효율적인 전역 상태 관리**  
Context의 복잡한 보일러플레이트를 넘어, Zustand의 단순하고 직관적인  
`create` 패턴으로 전역 스토어를 구축합니다. 다크 모드와 로그인 상태 등  
앱 전체에서 공유되는 데이터를 타입 안정성 있게 관리하고, `persist`  
미들웨어를 통해 새로고침 후에도 유지되는 사용자 경험을 구현합니다.

- **TanStack Query와 선언적 UI 패턴의 체득**  
`useEffect` 기반의 명령형 데이터 페칭에서 벗어나, `useQuery`를 이용한  
선언적 서버 상태 관리로 전환합니다. 캐싱(Caching)과 신선도(Stale) 개념을  
이해하고, `Suspense`와 `ErrorBoundary`를 결합하여 로딩과 에러 처리를  
컴포넌트 외부로 분리하는 간결하고 견고한 UI 설계 능력을 기릅니다.

- **고급 UX 최적화와 개발 도구 활용**  
`useTransition`을 활용해 데이터 로딩 중에도 UI 끊김 없는 부드러운  
카테고리 전환을 구현하며 사용자 경험(UX)을 극대화합니다. 또한, React  
Query DevTools를 통해 서버 데이터의 상태 변화를 실시간으로 관찰하고  
디버깅함으로써 데이터 흐름에 대한 깊은 통찰력을 확보합니다.
