## 실습 1 요구사항

**목표: `React.memo`를 활용한 불필요한 컴포넌트 리렌더링 방지**

**1. 성능 진단 및 문제 파악**
- 현재 `DeviceItem` 컴포넌트에는 `blockThread(50)`가 포함되어 있어 렌더링 시 의도적인 지연이 발생합니다.
- 상단 헤더의 **실시간 시계**가 1초마다 업데이트될 때마다, 상태가 변하지 않은 모든 `DeviceItem`들이  
불필요하게 리렌더링되며 메인 스레드를 점유하는 문제를 확인합니다.

**2. `React.memo` 적용 (Optimization)**
- `DeviceItem` 컴포넌트를 **`React.memo`** 로 감싸 고차 컴포넌트(HOC)로 만듭니다.
- **기대 효과**: 부모 컴포넌트(`SmartHomePanel`)가 시계 업데이트로 인해 리렌더링되더라도,  
`DeviceItem`의 Props(`name`, `status`, `onToggle`)가 변경되지 않았다면 리렌더링을 건너뛰도록 설정합니다.

**3. 렌더링 로그 확인**
- `console.log`를 통해 시계가 움직일 때마다 출력되던  
"🔌 [진단] 기기 리렌더링 발생..." 메시지가 더 이상 출력되지 않는지 확인합니다.

**4. UI 및 사용자 경험**
- `React.memo` 적용 전후의 스위치 애니메이션 부드러움을 비교합니다.
- 지연 시간(`delay`) 값을 높여보며 최적화의 중요성을 체감합니다.

## 실습 2 요구사항

**목표: `useCallback`을 통한 함수 참조 무결성 유지**

**1. 참조 동일성 문제 해결**
- `React.memo`를 적용했음에도 불구하고, 특정 기기의 스위치를 클릭하여  
`devices` 상태가 변경되면 **모든 기기 리스트가 다시 리렌더링**되는 현상을 발견합니다.
- 원인 분석: 부모가 리렌더링될 때마다 `toggleDevice` 함수가 새로 생성되어,  
자식에게 전달되는 `onToggle` Props의 참조값이 매번 바뀌기 때문임을 이해합니다.

**2. `useCallback` 적용 (Handler)**
- `toggleDevice` 함수를 **`useCallback`** 훅으로 감쌉니다.
- **의존성 배열(Dependency Array)** 을 빈 배열(`[]`)로 설정하여,  
컴포넌트가 처음 마운트될 때 생성된 함수 인스턴스를 계속 재사용하도록 합니다.
- **상태 업데이트 로직**: `setDevices` 내부에서 함수형 업데이트(`prev => ...`)를  
사용하여 의존성 배열에 `devices`를 넣지 않고도 최신 상태를 참조할 수 있도록 구현합니다.

**3. 최적화 결과 검증**
- 이제 특정 기기를 켰을 때, **클릭한 해당 기기만** 리렌더링 로그가 찍히고  
나머지 기기들은 정지 상태를 유지하는지 확인합니다.

## 실습 3 요구사항

**목표: `useMemo`를 활용한 무거운 연산 결과 재사용**

**1. 파생된 상태의 비용 계산**
- 현재 `connectedCount`는 `devices.filter().length`를 통해 매번 계산되고 있습니다.
- 시계가 1초마다 업데이트될 때마다 기기 목록의 변화가 없음에도 불구하고  
필터링 연산이 반복되는 낭비를 파악합니다.

**2. `useMemo` 적용 (Logic)**
- `connectedCount` 계산 로직을 **`useMemo`** 로 감쌉니다.
- **의존성 설정**: 오직 `devices` 배열이 변경될 때만 필터링 연산이  
다시 실행되도록 의존성 배열에 `[devices]`를 추가합니다.

**3. 효율적인 렌더링 구조 완성**
- 시계(`now`)가 바뀔 때는 `useMemo`에 의해 캐싱된 값을 즉시 반환하고,  
기기 상태(`status`)가 바뀔 때만 연산이 수행되는지 확인합니다.

**4. 접근성(Accessibility) 강화**
- 연결된 기기 개수가 표시되는 `<b>` 태그 주변에 **`aria-live="polite"`** 속성을  
유지하거나 추가하여, 기기 상태 변경으로 인해 숫자가 바뀔 때 스크린 리더 사용자가 즉시 알 수 있도록 합니다.
- `time` 태그의 `datetime` 속성을 활용하여 기계가 읽기 좋은 시간 형식을 제공합니다.
