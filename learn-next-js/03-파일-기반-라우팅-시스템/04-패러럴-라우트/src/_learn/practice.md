# 포켓몬 도감 'PokeLog' 라우팅 디자인: 병렬 라우트 편

Next.js의 App Router에서 하나의 레이아웃 안에 여러 개의 페이지를  
동시에 보여주는 **병렬 라우트(Parallel Routes)** 를 학습합니다.

대시보드 형태의 화면에서 '내 포켓몬 팀', '최근 본 포켓몬', '도감 통계' 등  
서로 다른 데이터를 가진 섹션들을 독립적인 슬롯(`@slot`)으로 나누어 관리하는 구조를 설계해 봅시다.

## 실습 폴더 구조

아래 구조에 따라 파일을 생성하세요.

```sh
/*
  app/
  ├── layout.tsx             # [전체 공통] 서비스 로고 및 메인 네비게이션
  ├── page.tsx               # [/] 홈 - "포켓로그에 오신 것을 환영합니다!"
  │
  ├── dashboard/             # [/dashboard] 통합 관리 화면 (병렬 라우트 중심)
  │   ├── layout.tsx         # ✨ @team, @stats 슬롯을 props로 받아 배치하는 핵심 레이아웃
  │   ├── page.tsx           # [/dashboard] 메인 컨텐츠 - "대시보드에 오신 것을 환영합니다!"
  │   │
  │   ├── @team/             # ✨ [슬롯 1] 내 포켓몬 팀 정보
  │   │   ├── page.tsx       # - "현재 구성된 6마리의 포켓몬 파티 목록"
  │   │   └── default.tsx    # - "슬롯 내 경로 불일치 시 보여줄 기본 화면"
  │   │
  │   └── @stats/            # ✨ [슬롯 2] 도감 수집 통계 정보
  │       ├── page.tsx       # - "전체 도감 완성률 및 타입별 수집 현황"
  │       └── default.tsx    # - "슬롯 내 경로 불일치 시 보여줄 기본 화면"
  │
  └── settings/              
      └── page.tsx           
*/
```
