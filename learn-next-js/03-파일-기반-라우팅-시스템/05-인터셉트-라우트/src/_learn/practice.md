# 포켓몬 도감 'PokeLog' 라우팅 디자인: 인터셉트 라우트 편

Next.js의 App Router에서 현재 페이지의 맥락을 유지하면서 다른 경로의 콘텐츠를  
가로채어 보여주는 **인터셉트 라우트(Intercepting Routes)** 를 학습합니다.

도감 목록에서 포켓몬을 클릭했을 때, 페이지 이동 없이 그 자리에서 상세 정보를  
모달(Modal)로 띄우고 새로고침 시에는 전체 페이지로 전환되는 구조를 설계해 봅시다.

## 실습 폴더 구조

아래 구조에 따라 파일을 생성하세요.

```sh
/*
  app/
  ├── layout.tsx             # [전체 공통] 서비스 로고 및 메인 네비게이션
  ├── page.tsx               # [/] 홈 - "포켓로그에 오신 것을 환영합니다!"
  │
  ├── pokemon/               # [/pokemon] 도감 목록 섹션
  │   ├── page.tsx           # - "전체 포켓몬 리스트 (클릭 시 인터셉트 발생)"
  │   │
  │   └── [id]/              # [/pokemon/25] 포켓몬 상세 페이지
  │       └── page.tsx       # - "새로고침이나 직접 접속 시 보이는 전체 상세 화면"
  │
  └── @modal/                # ✨ 병렬 라우트 슬롯을 이용한 모달 컨테이너
      ├── default.tsx        # - "모달이 닫혀있을 때 보여줄 빈(null) 화면"
      │
      └── (.)pokemon/[id]/   # ✨ [/pokemon/25] 경로 가로채기 (Intercepting)
          └── page.tsx       # - "목록에서 클릭 시 띄워줄 팝업/모달 상세 화면"
  │
  └── layout.tsx             # ✨ root layout에서 @modal 슬롯을 함께 렌더링
*/
```
