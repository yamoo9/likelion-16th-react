# 🗄️ Supabase SQL 스크립트 사용 가이드

이 디렉토리는 애플리케이션의 데이터베이스 설계, 데이터 관리, RLS 보안 설정 및 시스템 자동화를 위한 통합 SQL 가이드를 제공합니다.

## 🚀 권장 실행 순서

가장 안정적인 환경 구축을 위해 아래 번호 순서대로 스크립트를 실행하세요.

1.  **`01_create-table.sql`**: 테이블 구조 생성
2.  **`07_auto-profile-trigger.sql`**: **(중요)** 회원가입 시 프로필 자동 생성 설정
3.  **`08_auto-timestamp.sql`**: 수정 시간 자동 갱신 설정
4.  **`02_check-auth-user.sql`**: 내 계정의 UUID 확인 (가입 후 실행)
5.  **`03_update-data.sql`**: 데이터 소유권(user_id) 설정
6.  **`05_set-rls-policies.sql`**: 보안 정책(RLS) 적용
7.  **`09_storage-policies.sql`**: 파일 업로드 보안 설정 (이미지 사용 시)
8.  **`10_create-index.sql`**: 검색 및 정렬 성능 최적화

---

## 📄 파일별 상세 설명

### `01_create-table.sql` (테이블 생성)
*   **용도**: 테이블을 생성하고 기본 컬럼(`id`, `user_id`, `content` 등)을 정의합니다.
*   **특징**: 작성자 ID가 자동으로 기록되도록 `auth.uid()` 기본값을 설계에 반영합니다.

### `02_check-auth-user.sql` (사용자 확인)
*   **용도**: Supabase Auth에 가입된 사용자의 이메일과 UUID를 조회합니다.
*   **사용 시점**: 특정 사용자의 UUID를 복사하여 데이터 소유권을 수동으로 변경하거나 테스트할 때 사용합니다.

### `03_update-data.sql` (데이터 수정)
*   **용도**: 기존 데이터 내용을 수정하거나, `user_id`가 비어있는 데이터의 주인을 일괄 변경합니다.
*   **사용 시점**: 외부 데이터를 임포트한 후, 모든 데이터의 주인을 내 계정으로 설정하고 싶을 때 실행합니다.

### `04_cleanup-data.sql` (데이터 초기화)
*   **용도**: 테이블의 모든 데이터를 삭제하거나 특정 조건의 데이터를 제거합니다.
*   **주의**: `truncate` 명령은 복구가 불가능하므로 초기화 시에만 신중히 사용하세요.

### `05_set-rls-policies.sql` (보안 정책 설정)
*   **용도**: 행 레벨 보안(RLS)을 활성화하고, **"본인의 데이터만 읽고 쓰기"** 정책을 수립합니다.
*   **핵심**: `auth.uid() = user_id` 조건을 통해 타인의 데이터 접근을 원천 차단합니다.

### `06_manage-rls.sql` (정책 관리)
*   **용도**: 적용된 정책 목록 확인, 수정이 필요할 때 기존 정책 삭제(`drop`) 또는 비활성화합니다.

### `07_auto-profile-trigger.sql` (프로필 자동화)
*   **용도**: 사용자가 회원가입을 하면 자동으로 `profiles` 테이블에 기본 정보를 생성합니다.
*   **특징**: 소셜 로그인 정보(이름, 사진)를 자동으로 연동하여 사용자 경험을 향상시킵니다.

### `08_auto-timestamp.sql` (시간 자동화)
*   **용도**: 데이터가 수정(Update)될 때마다 `updated_at` 컬럼을 현재 시간으로 자동 갱신합니다.
*   **특징**: 개발자가 일일이 수정 시간을 코드로 업데이트할 필요가 없어 관리가 편리합니다.

### `09_storage-policies.sql` (파일 보안)
*   **용도**: Supabase Storage에 업로드되는 이미지나 파일에 대한 보안 정책을 설정합니다.
*   **핵심**: 본인이 업로드한 파일만 수정/삭제할 수 있도록 제한합니다.

### `10_create-index.sql` (성능 최적화)
*   **용도**: 데이터가 많아질 경우를 대비해 검색(제목, 내용) 및 정렬(최신순) 속도를 높입니다.
*   **사용 시점**: 서비스 규모가 커지거나 검색 속도가 느려졌을 때 적용합니다.

## 주의사항 및 팁

*   **실행 권한**: 트리거 및 함수 관련 스크립트(`07`, `08`)는 반드시 관리자 권한이 있는 **SQL Editor**에서 실행해야 합니다.
*   **UUID 입력**: `03_update-data.sql` 등에서 `'본인의_UUID_입력'` 부분은 실제 ID로 반드시 교체하세요.
*   **순서 엄수**: 특히 `07_auto-profile-trigger.sql`은 사용자가 가입하기 **전**에 미리 설정되어 있어야 정상 작동합니다.
*   **테스트**: 모든 보안 정책 적용 후에는 반드시 다른 계정으로 접속하여 보안이 잘 작동하는지 확인하세요.
